## Deployment Steps

### Backend Deploy
Render auto-deploys from GitHub `main` branch.

**Manual trigger:** Render Dashboard → inspection-api-o3hz → Deploy

**After deploy:**
```bash
# Verify health
curl https://inspection-api-o3hz.onrender.com/health
# Expected: {"status":"healthy","database":"connected"}

# Run migrations (only if schema changed)
# Render Dashboard → inspection-api-o3hz → Shell
flask db upgrade
```

### Web Frontend Deploy
Render auto-deploys from GitHub `main` branch.

**Build command (in render.yaml):**
```bash
pnpm install --no-frozen-lockfile
pnpm --filter @inspection/shared build
VITE_API_URL=https://inspection-api-o3hz.onrender.com pnpm --filter @inspection/web build
```

**After deploy:**
```bash
# Verify
curl -o /dev/null -w "%{http_code}" https://inspection-web.onrender.com
# Expected: 200
```

### Mobile App Build (EAS)
```bash
cd frontend/apps/mobile

# Preview build (internal testing)
eas build --platform all --profile preview

# Production build
eas build --platform all --profile production

# Submit to stores (after production build)
eas submit --platform ios --latest
eas submit --platform android --latest
```

**Required before EAS submit:**
- Update `eas.json` with real Apple ID, ASC App ID, Apple Team ID
- Add `google-services.json` for Android

## Rollback Procedure

### Backend Rollback
1. Render Dashboard → inspection-api-o3hz → Deploys
2. Find last working deploy
3. Click "Redeploy" on that commit

**If migration was run:** Contact database administrator — migrations may need manual rollback via `flask db downgrade <revision>`

### Web Frontend Rollback
1. Render Dashboard → inspection-web → Deploys
2. Click "Redeploy" on last working deploy

### Mobile Rollback
- iOS: App Store Connect → Activity → Previous build → Release previous version
- Android: Google Play → Release → Rollout → Halt + Rollout to previous track
